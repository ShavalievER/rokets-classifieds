# Попробовать server.js

## Текущая ситуация

- ✅ `server-safe.js` работает в диагностическом режиме
- ✅ Next.js `prepare()` завершается успешно
- ❌ Handler вызывает падение процесса

## Решение: Использовать улучшенный server.js

Я улучшил `server.js` с обработкой ошибок. Он должен показывать ошибки вместо падения.

### Шаг 1: Переключить на server.js

1. В **Setup Node.js App** измените **Application Startup File** на `server.js`
2. Нажмите **Save**
3. Нажмите **Stop**
4. Подождите 10 секунд
5. Нажмите **Start**
6. Подождите 30 секунд
7. Откройте `https://demo.rokets.delivery` в браузере

### Шаг 2: Проверить результат

Теперь должно быть одно из:
- ✅ Сайт работает (если handler работает)
- ❌ Страница с ошибкой (с деталями и stack trace)
- ❌ Internal Server Error (если процесс всё ещё падает)

### Если всё ещё Internal Server Error

Если `server.js` тоже показывает Internal Server Error, значит проблема на системном уровне:
- Ограничения памяти (LVE limits)
- Системные ограничения на shared hosting
- Проблемы с файлами .next

В этом случае нужно:
1. Проверить логи в cPanel
2. Обратиться в поддержку Verpex
3. Попробовать пересобрать проект
