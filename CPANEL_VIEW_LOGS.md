# Как открыть логи через File Manager в cPanel

## Способ 1: Через File Manager

### Шаг 1: Открыть File Manager

1. В cPanel найдите **File Manager**
2. Откройте его

### Шаг 2: Найти папку с логами

Логи обычно находятся в одном из этих мест:

**Вариант A: В корневой папке**
1. В File Manager перейдите в корневую папку (обычно `public_html/` или `home/ваш_username/`)
2. Найдите папку `logs/` или `logs/nodejs/`

**Вариант B: В домашней папке**
1. В File Manager нажмите на иконку **Home** (домик) в верхней панели
2. Найдите папку `logs/` или `logs/nodejs/`

**Вариант C: Через путь**
1. В File Manager введите в адресной строке путь: `~/logs/nodejs/`
2. Или: `~/logs/`

### Шаг 3: Найти файл лога

В папке `logs/nodejs/` найдите файл с именем вашего приложения:

- `demo.rokets.delivery.log`
- `nodejs.log`
- Или файл с текущей датой (например, `error_log-2026-01-06.log`)

### Шаг 4: Открыть файл

1. **Кликните правой кнопкой** на файл лога
2. Выберите **View** или **Edit**
3. Или просто **двойной клик** на файл

### Шаг 5: Просмотреть логи

Файл откроется в текстовом редакторе в браузере. Прокрутите вниз, чтобы увидеть последние записи.

## Способ 2: Через раздел Logs в cPanel

### Шаг 1: Найти раздел Logs

1. В cPanel найдите раздел **Logs**
2. Откройте его

### Шаг 2: Открыть Error Log

1. В разделе **Logs** найдите **Error Log**
2. Откройте его
3. Просмотрите последние ошибки

## Способ 3: Через Metrics → Errors

1. В cPanel найдите **Metrics**
2. Откройте **Errors**
3. Просмотрите последние ошибки

## Что искать в логах

### Успешный запуск

Ищите строки:
```
Starting Rokets classifieds...
Node.js version: v20.x.x
Port: XXXX
Base path: (none)
✅ Rokets classifieds ready on http://localhost:XXXX
```

### Ошибки

Ищите строки с:
- `Error:` - ошибки выполнения
- `Cannot find module` - отсутствующие модули
- `Failed to` - неудачные операции
- `EADDRINUSE` - порт занят
- `ENOENT` - файл не найден

### Запросы к серверу

После обновления `server.js` вы увидите строки:
```
[2026-01-06T...] GET /
[2026-01-06T...] GET /search
```

Это покажет, какие запросы приходят на сервер.

## Если файл лога не найден

### Вариант 1: Проверить другие места

Логи могут быть в:
- `~/logs/nodejs/`
- `~/logs/`
- `~/public_html/logs/`
- `~/error_log`

### Вариант 2: Создать тестовый запрос

1. Откройте сайт в браузере: `https://demo.rokets.delivery`
2. Подождите 10 секунд
3. Проверьте логи снова - должны появиться новые записи

### Вариант 3: Проверить настройки приложения

1. В **Setup Node.js App** проверьте, где сохраняются логи
2. Возможно, логи сохраняются в другом месте

## Копирование логов

Если нужно скопировать логи:

1. В File Manager откройте файл лога
2. Выделите нужный текст
3. Скопируйте (Ctrl+C)
4. Вставьте в текстовый редактор или отправьте в поддержку

## Полезные команды для поиска

Если есть доступ к терминалу или SSH:

```bash
# Найти все логи
find ~ -name "*.log" -type f

# Просмотреть последние 50 строк лога
tail -n 50 ~/logs/nodejs/demo.rokets.delivery.log

# Следить за логами в реальном времени
tail -f ~/logs/nodejs/demo.rokets.delivery.log
```

## Важно

⚠️ **Логи могут быть большими** - прокрутите вниз, чтобы увидеть последние записи.

⚠️ **Логи обновляются в реальном времени** - обновите страницу, чтобы увидеть новые записи.

⚠️ **Если файл очень большой** - используйте функцию поиска в редакторе (Ctrl+F), чтобы найти нужные строки.
