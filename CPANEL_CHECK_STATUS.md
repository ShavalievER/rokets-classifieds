# Как проверить статус и логи Node.js приложения в cPanel

## Проверка статуса приложения

### В Setup Node.js App

1. Откройте **Setup Node.js App** в cPanel
2. Найдите ваше приложение `demo.rokets.delivery`
3. Проверьте:
   - **Статус**: Должно быть **Running** (зелёный) или **Stopped** (красный)
   - **Порт**: Должен быть указан номер порта
   - **Node.js Version**: Должна быть версия 18+

## Просмотр логов

### Способ 1: Через File Manager

1. Откройте **File Manager** в cPanel
2. Перейдите в корневую папку (обычно `public_html/` или `home/ваш_username/`)
3. Найдите папку `logs/` или `logs/nodejs/`
4. Откройте файл с именем вашего приложения:
   - `demo.rokets.delivery.log`
   - Или `nodejs.log`
   - Или файл с текущей датой

### Способ 2: Через Error Log в cPanel

1. В cPanel найдите раздел **Logs**
2. Откройте **Error Log**
3. Проверьте последние ошибки

### Способ 3: Через Metrics → Errors

1. В cPanel найдите **Metrics**
2. Откройте **Errors**
3. Проверьте последние ошибки

### Способ 4: Через SSH (если есть доступ)

```bash
ssh ваш_username@rokets.delivery

# Проверить логи
tail -f ~/logs/nodejs/demo.rokets.delivery.log

# Или найти все логи
find ~ -name "*.log" -type f | grep nodejs

# Проверить, запущен ли процесс
ps aux | grep node
```

## Типичные проблемы и решения

### Проблема: Приложение в статусе "Stopped"

**Решение:**
1. В **Setup Node.js App** нажмите **Start**
2. Подождите 10-15 секунд
3. Проверьте статус снова

### Проблема: Приложение не запускается

**Проверьте:**
1. **Application Root** - правильный ли путь?
2. **Application Startup File** - правильный ли файл (`server.js`)?
3. **Node.js Version** - достаточно ли новая версия (18+)?
4. Зависимости установлены? (Run NPM Install)

### Проблема: В логах ошибка "Cannot find module"

**Решение:**
1. В **Setup Node.js App** нажмите **Run NPM Install**
2. Дождитесь завершения
3. Нажмите **Restart**

### Проблема: В логах ошибка ".next folder not found"

**Решение:**
1. Проверьте наличие папки `.next/` в File Manager
2. Включите показ скрытых файлов (Settings → Show Hidden Files)
3. Если папки нет - обновите репозиторий через Git Version Control

## Быстрая диагностика

### Шаг 1: Проверить статус
- ✅ Running = приложение запущено
- ❌ Stopped = нужно запустить

### Шаг 2: Проверить файлы
- ✅ `server.js` существует?
- ✅ `package.json` существует?
- ✅ `.next/` существует? (скрытая папка)
- ✅ `node_modules/` существует?

### Шаг 3: Проверить конфигурацию
- ✅ Application Root правильный?
- ✅ Application Startup File правильный?
- ✅ Node.js Version правильная?

### Шаг 4: Перезапустить
1. Нажмите **Stop**
2. Подождите 5 секунд
3. Нажмите **Start**
4. Подождите 10-15 секунд

## Если ничего не помогает

1. Удалите приложение в **Setup Node.js App**
2. Создайте новое с правильными настройками
3. Установите зависимости (**Run NPM Install**)
4. Запустите приложение (**Start**)
