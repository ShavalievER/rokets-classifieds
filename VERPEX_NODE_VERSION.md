# Критично: Обновление Node.js на Verpex

## Проблема

Ваше приложение использует **Node.js 10.24.1**, но Next.js 15 требует **Node.js 18+**.

Ошибка "Script exit code: 1" возникает из-за несовместимости версий.

## Решение: Обновить Node.js до 18+

### Шаг 1: Обновить версию Node.js в cPanel

1. Зайдите в **cPanel → Setup Node.js App**
2. Найдите ваше приложение для `demo.rokets.delivery`
3. Нажмите **Edit** (карандаш)
4. Найдите поле **"Node.js version"** или **"Runtime version"**
5. Выберите **Node.js 18.x** или **20.x** (самую новую доступную)
6. Нажмите **Save**

### Шаг 2: Переустановить зависимости

После обновления Node.js:

1. В настройках Node.js App нажмите **Run npm install** (или `npm install --legacy-peer-deps`)
2. Дождитесь завершения установки
3. Нажмите **Run npm run build**
4. После сборки нажмите **Restart**

### Шаг 3: Проверить логи

После перезапуска:

1. Откройте **Error Logs** в Node.js App
2. Должны увидеть:
   ```
   ✅ Rokets classifieds ready on http://localhost:PORT
   ```
3. Если видите ошибки — скопируйте их сюда

## Альтернатива: Использовать стандартный Next.js start

Если обновление Node.js невозможно, можно попробовать использовать стандартный `next start`:

1. В **Setup Node.js App → Edit**:
   - **Startup file**: оставьте пустым или удалите
   - **Start Command**: `npm start`
   - (Next.js автоматически использует `.next` папку после `npm run build`)

Но это **не рекомендуется**, так как Next.js 15 всё равно требует Node.js 18+.

## Проверка версии

После обновления в логах должно быть:
```
Node.js version: v18.x.x
```
или
```
Node.js version: v20.x.x
```

Если всё ещё видите `v10.24.1` — версия не обновилась, попробуйте:
1. Удалить приложение
2. Создать заново с Node.js 18+
