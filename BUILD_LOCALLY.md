# Инструкция: Сборка проекта локально для Verpex

## Проблема

На shared хостинге Verpex не хватает памяти для сборки Next.js 15. Решение: собрать локально и загрузить готовую папку `.next`.

## Шаги

### 1. Собери проект локально

На своём компьютере:

```bash
cd commerce
npm install --legacy-peer-deps
npm run build
```

После успешной сборки появится папка `.next/`

### 2. Загрузи папку `.next` на Verpex

**Вариант A: Через File Manager в cPanel**

1. Зайди в **File Manager**
2. Перейди в `repositories/rokets-classifieds/commerce/`
3. Загрузи папку `.next` (можно заархивировать в `.zip`, загрузить и распаковать)

**Вариант B: Через FTP**

1. Подключись к серверу через FTP клиент (FileZilla, WinSCP и т.д.)
2. Загрузи папку `.next` в `repositories/rokets-classifieds/commerce/`

### 3. На Verpex запусти приложение БЕЗ сборки

В **Setup Node.js App**:

1. **НЕ нажимай** "Run npm run build" (сборка уже готова)
2. Просто нажми **Restart** приложения
3. Приложение должно запуститься с готовой сборкой

### 4. Проверь

Открой `https://demo.rokets.delivery` - должен открыться классифайд!

## Важно

- После каждого обновления кода нужно будет собирать локально и загружать `.next` заново
- Или попросить Verpex увеличить лимиты памяти для сборки
